# 일반적인 프로그래밍 원칙

<br>
## 57. 지역변수의 범위를 최소화하라.

이 원칙은 클래스와 멤버의 접근 권한을 최소화하라는 것과 비슷하다. **지역변수의 유효범위를 최소로 줄이면 코드 가독성과 유지보수성이 높아지고 오류 가능성은 낮아진다.**
